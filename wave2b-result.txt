Wave 2B — Intelligence API Response Fix — COMPLETED

## Problem
Server intelligence routes wrap response arrays/objects in wrapper objects:
  - GET /api/intelligence/priorities  → { priorities: [...] }
  - GET /api/intelligence/business    → { businessIntelligence: {...} }
  - GET /api/intelligence/stats       → { stats: [...] }

But the React Query hooks in useBusinessIntelligence.ts returned `res.json()` directly,
so the client received `{ priorities: [...] }` instead of the bare array, causing
`.map()` runtime errors when the dashboard tried to iterate.

## Fix Applied
File: client/src/hooks/useBusinessIntelligence.ts

Three hooks were updated to unwrap the server response:

1. useBusinessIntelligence (line ~91):
   - Before: `return res.json();`
   - After:  `const data = await res.json(); return data.businessIntelligence ?? data;`

2. useProductPriorities (line ~125):
   - Before: `return res.json();`
   - After:  `const data = await res.json(); return data.priorities ?? data;`

3. useProductStats (line ~194):
   - Before: `return res.json();`
   - After:  `const data = await res.json(); return data.stats ?? data;`

The `?? data` fallback ensures backwards compatibility if the server response format
ever changes to return bare arrays/objects directly.
